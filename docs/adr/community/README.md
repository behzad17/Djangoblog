# Community Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) for the Peyvand Community module.

ADRs document significant architectural choices, their context, and consequences. Each ADR is numbered sequentially and should be updated to **Accepted** once the decision is finalized.

## Index

| ADR | Title | Status |
|-----|-------|--------|
| [ADR-001](ADR-001-community-categories.md) | Community has its own categories | Proposed |
| [ADR-002](ADR-002-flat-replies.md) | Replies are flat | Proposed |
| [ADR-003](ADR-003-discussion-status.md) | Discussion status: Open / Closed / Hidden (MVP moderation via hidden) | Accepted (MVP) |
| [ADR-004](ADR-004-separate-from-askme.md) | Community is separate from AskMe | Proposed |
| [ADR-005](ADR-005-soft-delete.md) | Soft delete for discussions | Proposed |
| [ADR-006](ADR-006-denormalized-counters.md) | Denormalized counters | Proposed |
| [ADR-007](ADR-007-configurable-trust-threshold.md) | Configurable moderation trust threshold | Proposed |
| [ADR-008](ADR-008-search-strategy.md) | Search strategy | Proposed |
| [ADR-009](ADR-009-url-namespace.md) | URL namespace under `/community/discussions/` | Proposed |
| [ADR-010](ADR-010-community-notifications.md) | Dedicated community notification preference | Proposed |

## How to write an ADR

Copy [template.md](template.md) and fill in each section before marking an ADR as **Accepted**.
